API REST Python

Dans le cadre du cours Python, interrogation et exposition d'APIs REST avec les bibliothèques requests et Flask : requêtes HTTP, parsing JSON et création d'endpoints.

ContexteProjet scolaire

Consommation d'API REST en Python

Présentation

Ce projet introduit la consommation et la création d'APIs REST en Python. La bibliothèque requests est utilisée pour effectuer des requêtes HTTP (GET, POST…) vers des APIs externes, récupérer des données JSON et les traiter côté script.

Flask permet quant à lui d'exposer ses propres endpoints REST : définir des routes, retourner du JSON structuré et gérer les méthodes HTTP. Le projet couvre ainsi les deux faces d'une architecture orientée services.

python — api_client.py
$ python api_client.py
# GET /api/data → 200 OK
{ "status": "success",
  "items": ["Paris", "Tokyo", "New York"],
  "count": 3 }
$ flask run --port 5000
# Running on http://127.0.0.1:5000
✔ Endpoints enregistrés : /api/items, /api/items/<id>

Ce que couvre le projet

Requêtes HTTP

Utilisation de requests.get() et requests.post() pour interroger des APIs externes et récupérer des réponses JSON.

Parsing JSON

Désérialisation des réponses avec .json(), navigation dans les structures imbriquées et extraction des données utiles.

Création d'endpoints

Définition de routes Flask (@app.route), gestion des méthodes GET/POST et retour de réponses JSON avec jsonify().

Gestion des erreurs

Vérification des codes HTTP (status_code), gestion des exceptions réseau et retours d'erreurs structurés côté serveur.

Technologies & compétences

Le projet mobilise Python 3 dans un contexte backend pur : communication réseau, manipulation de données structurées et exposition de services web légers. Ces bases sont directement réutilisables dans des projets plus complexes (microservices, automatisation, scraping, intégrations tierces).
