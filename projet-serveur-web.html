Serveur Web

Montage et configuration complète d'un serveur web sous Linux pour héberger et servir des fichiers statiques et des applications web dans un cadre scolaire.

ContexteProjet scolaire
RôleAdministration système
EnvironnementLinux / Apache / Nginx

Configuration du serveur web

Présentation

Ce projet d'administration système consistait à monter, configurer et mettre en service un serveur web capable d'héberger des fichiers statiques (HTML, CSS, images) et d'exposer des applications web sur le réseau.

L'objectif était de comprendre l'ensemble de la chaîne : installation du système d'exploitation, configuration du serveur HTTP, gestion des droits et sécurisation de l'accès, puis déploiement effectif de contenus.

Étapes de mise en place

  1. Installation de l'OS — Déploiement d'une distribution Linux (Debian/Ubuntu) sur la machine serveur ou dans une VM.
  2. Installation du serveur HTTP — Mise en place d'Apache ou Nginx, activation des modules nécessaires.
  3. Configuration des virtual hosts — Définition des hôtes virtuels pour servir plusieurs sites depuis un même serveur.
  4. Gestion des droits — Attribution des permissions fichiers, configuration des utilisateurs et groupes système.
  5. Déploiement des fichiers — Transfert et organisation des fichiers HTML/CSS dans les répertoires web.
  6. Test & vérification — Validation de l'accès réseau, logs d'erreur et de débogage.

Commandes clés

Quelques commandes utilisées lors de la configuration du serveur :

# Installation Apache
sudo apt install apache2

# Activation d'un virtual host
sudo a2ensite mon-site.conf
sudo systemctl reload apache2

# Vérification du statut
sudo systemctl status apache2

# Droits sur le répertoire web
sudo chown -R www-data:www-data /var/www/html

Compétences développées

Administration Linux

Navigation en ligne de commande, gestion des paquets avec apt, droits et permissions système.

Configuration serveur

Mise en place d'Apache/Nginx, virtual hosts, fichiers de configuration et rechargement de service.

Réseau & hébergement

Compréhension du modèle client-serveur, ports HTTP/HTTPS, résolution DNS locale.

Débogage

Lecture des logs Apache (access.log, error.log), identification et correction des erreurs de configuration.

Bonnes pratiques appliquées

  • Séparation des configurations — Un fichier de configuration par site dans /etc/apache2/sites-available/.
  • Gestion des droits minimaux — Principe du moindre privilège appliqué aux fichiers web.
  • Logs activés — Suivi des accès et erreurs pour faciliter le débogage.
  • Test en local d'abord — Validation en réseau local avant toute exposition externe.

Bilan personnel

Comprendre ce qui se passe côté serveur — de l'installation du daemon HTTP à la livraison d'une page au navigateur — est fondamental pour tout développeur ou administrateur système. Ces bases sont directement réutilisables pour déployer des projets web en production ou configurer des environnements de développement local.
